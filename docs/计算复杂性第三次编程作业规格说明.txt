任务  1 ： MAX-3SAT 的随机近似算法
MAX-3SAT 问题的输入是一个  3SAT 布尔公式，目标是找到一组变量赋值，使得被满足的子句数
量尽可能多。
本任务中，同学们需要实现如下随机近似算法：
循环 : 
    随机生成一组变量赋值 
    计算该赋值满足的子句数量 
直到 : 
    满足的子句数量不少于全部子句数量的  7/8 
输出 : 
    该组变量赋值、满足的子句数量、循环次数和运行时间
设输入公式共有   个子句，则程序应当持续随机生成变量赋值，直到找到一组满足至少  
个子句的赋值。

任务  2 ： METRIC-TSP 的  2- 近似算法

===== 第4页 =====
METRIC-TSP 问题的输入是一个满足三角不等式的带权完全图，目标是找到一条总权重尽可能小
的回路，使得每个顶点都被访问至少一次。
本任务中，同学们需要实现课件  Approximation Algorithms.pdf 中基于最小生成树的  2- 近似算
法：
输入 : 
    带权完全图  G = (V, E) ，边权  c 满足三角不等式 
算法 : 
    1. 计算  G 的最小生成树  MST 
    2. 任选一个顶点  s 作为根 
    3. 从  s 开始对  MST 进行  DFS 遍历 
    4. 根据  DFS 遍历顺序构造  TSP 近似回路 
    5. 输出该回路及其总代价、最优解代价和近似比例 
输出 : 
    一条  METRIC-TSP 近似回路
程序需要输出最小生成树  MST 、 DFS 遍历顺序、 TSP 近似回路、 MST 代价、 TSP 近似回路代价、
最优  TSP 回路代价、近似比例和运行时间。
METRIC-TSP 输入格式
为便于生成满足三角不等式的实例，本次作业统一采用二维坐标点作为输入。程序根据点之间的
欧氏距离自动构造带权完全图。
一个  txt 文件代表一个  METRIC-TSP 测试用例，示例如下：
a 0 0 
b 2 3 
c 5 1 
d 6 5 
e 8 2 
f 1 6
格式说明如下：
1. 每一行表示一个城市，即一个顶点；
2. 每一行包含城市名称、横坐标、纵坐标三项；
3. 城市名称与坐标之间以空格分隔；
4. 输入实例至少包含  6 个城市；
5. 不同城市不应具有完全相同的坐标。

METRIC-TSP 近似算法实现要求
程序应当完整实现以下步骤：
1. 构造带权完全图
根据输入城市坐标，计算任意两个城市之间的边权，构造带权完全图。
2. 计算最小生成树  MST 
使用  Prim 算法或  Kruskal 算法计算  MST 。同学们可以自行选择实现方式，但不能直接调用第
三方库。
3. DFS 遍历  MST 
从根节点开始对  MST 进行  DFS 遍历。
4. 构造  TSP 近似回路 
根据  DFS 首次访问顶点的顺序构造回路，并在最后返回起点。例如：
DFS 遍历顺序： a -> b -> c -> d 
TSP 回路： a -> b -> c -> d -> a
5. 计算  TSP 近似回路总代价 
根据原始完全图中的边权计算该  TSP 回路的总代价。
6. 计算最优  TSP 回路总代价 
为了给出近似比例，本次作业要求对测试用例计算最优  TSP 回路总代价。测试用例城市数量
不宜过大，建议不超过  9 个城市。
